# Design tokens — Job Management Dashboard

Direction codename: **Drafting Table**. Engineering-precise, calm, editorial-meets-technical. Inspired by drafting paper, Linear's restraint, and Bloomberg-terminal data density without the ugly. The signature moves are (1) the per-status geometric glyph, (2) Fraunces serif headlines paired with IBM Plex Sans/Mono body+data, (3) a faint paper-grid background that hints at the engineering context without dominating.

Open `docs/design/preview.html` in a browser to see the reference. This file is the extract for React + Tailwind v4 implementation.

## Brand alignment (Rescale)

The neutrals and the accent blue are sampled directly from the Rescale logo at `frontend/public/rescale-logo.png` so the dashboard reads as a Rescale-internal product.

- **Ink**: `#1B1B1B` — the logo's wordmark + cloud-outline black. Slightly less warm than `stone-900`; chosen for brand fidelity.
- **Accent**: `#489ABD` — the logo's blue squares. A muted steel blue, deliberately desaturated. Used for links, focus rings, and selected-state accents.
- **Accent (soft)**: `#E8F2F8` — for hover halos / subtle backgrounds tied to the accent family.
- **Accent (strong)**: `#336F8B` — darker brand blue for filled-CTA hover states (none in v1; reserved).

The cloud-mark + pixelated-square motif of the logo also dovetails with the geometric per-status glyphs — the visual language is internally consistent.

### Logo placement (locked)

- **List page header**: full lockup (mark + wordmark) at `h-10` (~40 px) on the left, label `OPS · INTERNAL` and version on the right, separated from the page title by a 1px rule.
- **Detail page top bar**: same lockup at `h-8` (~32 px) on the right, `opacity-90`, paired with the back link on the left.
- **Footer**: not used (avoid double branding; the header carries it).
- **Single-source-of-truth**: `frontend/public/rescale-logo.png` (also mirrored to `docs/design/assets/` so the preview HTML is self-contained).

## Typography

Three families, each with one job. Keep the pairing strict.

| Token | Family | Use |
| --- | --- | --- |
| `font-display` | **Fraunces** (variable, opsz 9–144, weight 400/500/600) | Page titles, section headings. Distinctive, warm, editorial. |
| `font-sans` | **IBM Plex Sans** (400/500/600) | Body, UI labels, buttons. Engineered feel without being cold. |
| `font-mono` | **IBM Plex Mono** (400/500) | IDs, timestamps, status labels, technical data. |

### Status palette

Each state has a **color + geometric glyph** so it reads with or without color.

| State | Glyph | Background | Text | Border |
| --- | --- | --- | --- | --- |
| PENDING | open circle (`○`) | `slate-100` `#F1F5F9` | `slate-700` `#334155` | `slate-300` `#CBD5E1` |
| RUNNING | filled triangle / play (`▶`) | `amber-100` `#FEF3C7` | `amber-800` `#92400E` | `amber-300` `#FCD34D` |
| COMPLETED | filled circle (`●`) | `emerald-100` `#D1FAE5` | `emerald-800` `#065F46` | `emerald-300` `#6EE7B7` |
| FAILED | cross (`✕`) | `red-100` `#FEE2E2` | `red-800` `#991B1B` | `red-300` `#FCA5A5` |

The glyphs are tiny inline SVGs (10×10 viewbox). See `preview.html` for the exact SVG paths. They live as a `<StatusPill state="...">` component that takes one of the four values.

## Spacing rhythm

4px baseline. Components on **8 / 16 / 24 / 32** multiples.

| Token | px | Use |
| --- | --- | --- |
| Tight | 4 | Inline gaps, pill internals |
| Snug | 8 | Form gap, button padding-Y |
| Medium | 12–16 | Card padding-X, button padding-X |
| Loose | 24–32 | Section spacing, header margins |
| Roomy | 48–64 | Empty-state padding, page rhythm |

Page max-width: **1100px**, side padding **32px** (`px-8`). All content lives in this column; no full-bleed.

## Component primitives

These are the building blocks the React implementation should produce.

### `StatusPill`

Props: `state: 'PENDING' | 'RUNNING' | 'COMPLETED' | 'FAILED'`. Renders a glyph + uppercase mono label, color per state. Used inline in the table, in the detail header, and in the timeline markers.

### `Button`

Three variants:
- **primary** — black bg, white text, used for the create / apply CTAs
- **ghost** — transparent, hover gets a faint stone bg, used for pagination / secondary
- **danger** — outline style, red text, hover gets a tint. Solid red bg only inside the delete-confirm modal.

### `Input`

White bg, 1px stone border. Focus ring: 3px `blue-700/15` halo + accent border. Error variant flips border to `red-600` and shows a mono error message below.

### `Panel`

Card surface: white bg, 1px stone border, 6px radius, very subtle two-layer shadow:

```css
box-shadow:
  0 1px 0   rgba(28,25,23,0.02),
  0 1px 3px rgba(28,25,23,0.04);
```

### `Label`

10.5px Plex Sans, weight 500, uppercase, letter-spacing 0.14em, `stone-500` color. Used as a section marker (above page titles, beside form fields, as table column headers).

### `FilterPill`

12px Plex Sans 500, 4px radius, 4×10 padding. Active state is solid stone-900 bg + white text. Idle is muted text on transparent; hover gets a stone-100 bg.

### `Timeline`

A vertical rail (1px stone line) with markers at each event. Each marker is a 19px round white "well" centered on the rail with the status glyph inside, drawn from the `StatusPill`'s glyph SVG.

### `Pagination`

Two ghost buttons (`← Previous` / `Next →`). Disabled state at 35% opacity, no cursor change since cursor pagination doesn't always reveal the bound. Mono pagination caption to the left ("Cursor pagination · newest first").

## Page-level patterns

### List page (`/jobs`)

Header band → primary panel containing { create form bar, filter row, table, pagination }. Below the panel, supplementary states (validation error, empty state) demonstrate variants the component must support.

### Detail page (`/jobs/:id`)

Back link → header (label + ID, big display name, current status pill, created/updated timestamps) → action panel (status update buttons + apply/delete) → history timeline panel.

The delete-confirm dialog is a standalone panel (could be a modal or an inline expand) reusing the same Panel/Button primitives.

## Implementation notes for the React stream

- Use Tailwind v4 inline color values for status palettes (`bg-amber-100 text-amber-800 border-amber-300`) rather than custom theme tokens — keeps the component code readable and matches Tailwind's idiom.
- Define the three font-families in `tailwind.config.ts` (`fontFamily: { display: ['Fraunces', ...], sans: ['IBM Plex Sans', ...], mono: ['IBM Plex Mono', ...] }`) and pull from the same Google Fonts link.
- The four status glyphs are best as a single `StatusPill` component switching on the prop. Don't inline the SVG paths in every consumer.
- Keep the paper-grid background as a single utility class (e.g. `.paper-grid` in a small `index.css`) — it's awkward to reproduce inline in Tailwind v4.
- Register the three custom Rescale colors in `tailwind.config.ts` under `theme.extend.colors.rescale = { ink: '#1B1B1B', blue: '#489ABD', 'blue-soft': '#E8F2F8', 'blue-strong': '#336F8B' }` so component code can use `bg-rescale-blue` / `text-rescale-ink` cleanly.
- Logo lives at `/rescale-logo.png` (Vite serves `public/` at root). Import with `<img src="/rescale-logo.png" alt="Rescale" />` — no bundler import.
- Favicon links go in `frontend/index.html` (Vite's entry HTML), not in code.

## What's deliberately NOT in this design

- Dark mode (out of scope per the brief)
- Loading skeletons (TanStack Query handles this; we'll use a simple muted-text "Loading…" first pass)
- Toast notifications (left for the React stream to decide; an inline banner pattern would fit cleanly within the existing Panel primitive)
- Mobile breakpoints (the brief says ≥768px is enough; the existing layout starts to feel cramped under 700px but doesn't break)
